Reliable Low-Latency UDP Streaming System

Overview

This project implements a real-time media streaming system over an unreliable UDP network. The objective is to deliver audio frames generated every 20 milliseconds while minimizing playout delay and maintaining reliable playback despite packet loss, delay variation, reordering, and duplicate packets.

Harness Player

The relay simulates an unreliable network by introducing packet loss, random delays, packet duplication, and packet reordering.

Sender Design

  • Receives audio frames from the harness source.
  • Extracts the sequence number and payload.
  • Constructs a network packet.
  • Sends packets to the relay using UDP.
  • The sender is designed for low latency while keeping bandwidth usage within the required limits.

Receiver Design

  • Receives packets from the relay.
  • Stores packets according to their sequence number.
  • Removes duplicate packets.
  • Maintains playback timing using a jitter buffer.
  • Forwards packets to the harness player every 20 milliseconds.
  • If a packet is unavailable at playback time, a silence frame is transmitted instead.

Design Choices

  • UDP is used because it provides low transmission latency.
  • Sequence numbers enable ordered playback.
  • Duplicate packets are discarded to prevent repeated playback.
  • A jitter buffer smooths packet arrival variations.
  • Missing packets are replaced with silence to maintain continuous playback.
  • The implementation prioritizes real-time performance over perfect reliability.

Trade-Offs

Higher Delay Lower Delay
Fewer missed packets Lower latency
Larger jitter buffer Greater risk of deadline misses
More stable playback Better responsiveness

System-Level Operation

The source generates one audio frame every 20 milliseconds. The sender immediately forwards each frame to the relay using UDP. The relay introduces network impairments including delay, packet loss, duplication, and reordering. The receiver stores incoming packets inside a buffer indexed by sequence number. At fixed 20-millisecond intervals, the receiver retrieves the expected frame and forwards it to the harness player. If the expected frame has not arrived before its playback deadline, the receiver transmits a silence frame instead. This approach preserves continuous playback while minimizing interruptions caused by network instability.

Conclusion

This project demonstrates a lightweight real-time transport system capable of handling unreliable network conditions using UDP communication, packet ordering, duplicate suppression, and jitter buffering. The overall design balances latency, bandwidth consumption, and playback reliability while satisfying the assignment's requirements.
